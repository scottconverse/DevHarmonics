MORE THAN A CHAT WRAPPER

Agents are useful. A coordinated, inspectable system is better.

01

Use the subscriptions you already have

Provider authentication stays inside official Codex, Claude Code, and Antigravity login flows. DevHarmonics never collects your passwords or requires model API keys.

02

Parallelize without agents colliding

A typed dependency graph determines what is ready. Every task receives its own Git branch and worktree before it can edit.

03

Demand evidence, not confidence

Allowlisted validators produce durable receipts. Failed output goes back to workers for bounded retries; a final reviewer judges the integrated result.

THE RUN LOOP

From objective to reviewable branch.

  1. 01
    PlanA read-only architect converts the objective into a validated dependency graph.
  2. 02
    DispatchReady tasks go to the best available provider, at the concurrency you or the architect choose.
  3. 03
    VerifyConfigured checks run in isolated worktrees. Exact failures feed bounded retries.
  4. 04
    IntegratePassing commits merge serially into dedicated repository branches and one exact integration set.
  5. 05
    ReviewA configured read-only reviewer quorum returns a verdict while SQLite preserves the trail.
