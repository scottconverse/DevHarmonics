What do these run statuses mean?

Planning: the AI is still building the plan — nothing has changed yet. Awaiting approval: the run is stopped and needs your click above before it continues. Running: AI workers are actively making changes, inside the limits you approved. Paused: work is stopped; resuming starts a fresh recovery run. Ready: the run finished and its work is ready to deliver. Not ready: the run finished but something didn't pass — a task, a check, or the independent review — open the task cards and Final review below to see which. Failed: the run itself could not complete. Cancelled: the run was stopped before finishing.

DECISION RECORDS

Decisions

A decision record captures a question the team faced, every option it weighed, which one was chosen, and — just as important — every option that was rejected and why. Rejections matter: without them, a killed approach gets re-proposed and re-argued from scratch at full cost. Recording it once means it never has to be re-litigated.

AI MODELS

Known models

Nothing here spends money or changes code by itself.

Known models: every model DevHarmonics has seen from a connected provider or you added manually (excludes retired models). Passed a role check: has passed at least one check proving it can do a specific job (planning, working, or reviewing). Ready to be used: turned on, currently qualified, and eligible to be picked for a run right now. Estimated local capacity: a rough guess at how many local-model tasks can run at once on this machine — an estimate DevHarmonics uses for planning, not something it enforces.

Qualifying runs a small test task through the model to confirm it can safely do a specific job before DevHarmonics will use it. Higher thinking effort can improve results on hard tasks but costs more and is slower. Model info is "verified" when it comes from the provider's own published specs, or "inferred" when DevHarmonics estimated it from limited data.

Status meanings: Not checked yet — never tested · Needs re-checking — the provider changed the model since it last passed · Excluded — you turned this model off on purpose · Needs benchmark — needs an extra accuracy test before it can be used · Not qualified — failed, or hasn't passed a role check yet · Qualified (inactive) — passed its check but turned off · Active — DevHarmonics can use this model now.
